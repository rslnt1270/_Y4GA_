**El error: canvas buffer demasiado pequeño vs. tamaño CSS**

El problema era este:

```html
<!-- HTML fija el buffer interno en 480×480 píxeles -->
<canvas id="board" width="480" height="480"></canvas>
```

```css
/* CSS lo muestra a ~850-900px en desktop */
#board { width: min(calc(100vw - 340px), calc(100vh - 50px), 900px); }
```

El browser escalaba 480px → 900px = ×1.875 de magnificación → **pixelado/borroso**.

**Por qué pasa:** El `<canvas>` tiene dos tamaños independientes:
- **Buffer interno** (`canvas.width` / `canvas.height`) — píxeles reales de dibujado
- **Tamaño CSS** — cuánto ocupa en pantalla

Cuando el CSS > buffer, el browser hace upscale bilineal = blur.

**El fix ya está en el código** (líneas 676-683):

```js
const dpr     = window.devicePixelRatio || 1;
const cssSize = canvas.offsetWidth || 480;   // tamaño real CSS
const bufSize = Math.round(cssSize * dpr);   // píxeles físicos
canvas.width  = bufSize;   // buffer = ~900px × 2 = 1800px en Retina
canvas.height = bufSize;
ctx.scale(bufSize / 480, bufSize / 480);    // lógica sigue en coords 480
```
